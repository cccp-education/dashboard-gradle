= Session 010 — Dashboard site JBake-compatible pour serveur bakery
:date: 2026-06-23
:status: TERMINÉ

== Objectif

Permettre à runner-gradle/bakery-gradle de servir le dashboard via sa tâche `serve`,
en transformant l'output du plugin dashboard en site JBake valide.

== Décisions

* Le dashboard émet désormais *deux* structures dans `build/dashboard` :
  - site plat (legacy) : `index.html` + `styles.css` + `dashboard-data.json`
  - site JBake : `jbake/` avec `jbake.properties`, `content/index.html`, `templates/index.thyme`,
    `assets/css/styles.css`, `assets/js/dashboard-data.json`
* `publishDashboard` est renommé en `publishDashboardSite` pour éviter l'ambigüité
  avec les tâches Maven `publishDashboardPluginMarker...` du plugin `maven-publish`.
* `publishDashboardSite` publie le sous-répertoire `jbake/` à plat dans `build/dashboard-publish`.
* Le template JBake `index.thyme` est minimal (`[(${content.body})]`) pour ne pas écraser le HTML.
* `site.yml` de `office/sites/dashboard/` pointe vers `dashboard-gradle/build/dashboard-publish`.
* Le CSS est référencé sous `css/styles.css` dans le contenu JBake car JBake déplace
  `assets/css/...` vers `css/...` dans le répertoire de sortie.
